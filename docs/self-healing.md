# Self-Healing Wiki

LLMBase detects and repairs its own issues automatically. The `auto_fix` pipeline runs 7 steps in order.

## The Pipeline

```
auto_fix()
  1. clean_garbage      → Remove template stubs, CJK slugs, prompt leaks
  2. fix_dirty_tags     → Strip LLM prompt text from tag fields
  3. normalize_tags     → LLM merges synonymous tags (e.g., confucian-philosophy → confucianism)
  4. fix_metadata       → LLM generates missing summary/tags
  5. fix_broken_links   → Alias-aware; only creates stubs for truly missing concepts
  6. merge_duplicates   → CJK-aware dedup with content merge (叠加进化)
  7. fix_uncategorized  → Regenerate taxonomy for unclassified articles
```

## Trigger Methods

| Method | Command |
|--------|---------|
| **CLI** | `llmbase lint heal` |
| **Web UI** | Health page → "Auto Fix" button |
| **API** | `POST /api/lint/fix` |
| **Worker** | Automatic every 24h (configurable) |

## What Each Step Does

### 1. Clean Garbage

Detects and removes:
- Unfilled LLM templates (`"English Title / 中文标题"`)
- Summary contains prompt leak (`"The user says..."`)
- CJK slugs (filename is Chinese instead of pinyin)
- Near-empty articles (< 50 characters)

### 2. Fix Dirty Tags

Tags containing LLM prompt text like `"2-4 tags. we need to interpret..."` are stripped. If too few clean tags remain, LLM regenerates them.

### 3. Normalize Tags

LLM analyzes all tags and groups synonyms:

```
confucian-philosophy → confucianism
daoist-philosophy → daoism
buddhist-ethics → buddhist-ethics (NOT merged — different concept)
```

### 4. Fix Metadata

Articles missing `summary` or `tags` get them generated by LLM.

### 5. Fix Broken Links

Uses the alias system to resolve `[[参禅]]` → `can-chan.md`. Only creates stub articles for links that truly have no matching article after alias resolution (including simplified/traditional conversion).

### 6. Merge Duplicates

Pre-filter (no LLM needed):
- CJK title substring matching (`仁` in title → matches `仁爱`)
- Slug substring (≥4 chars for ASCII, any length for CJK)
- Tag Jaccard similarity (≥60%)
- Simplified ↔ traditional expansion

Merge:
- ASCII slug preferred over CJK slug as primary
- Content merged section-by-section (keeps longer version per language)
- Tags merged (union)
- Sources merged (deduped by key)
- All `[[wiki-links]]` rewritten to point to primary

### 7. Taxonomy Regeneration

LLM-generated emergent categories. For large KBs (100+ articles), uses two-phase approach to avoid token overflow.

## Health Checks (8 categories)

```
llmbase lint check
```

| Check | What it finds |
|-------|---------------|
| `structural` | Missing index files |
| `broken_links` | Wiki-links with no matching article (alias-aware) |
| `orphans` | Articles with no incoming links |
| `missing_metadata` | Missing title, summary, or tags |
| `dirty_tags` | LLM prompt leaks in tag fields |
| `duplicates` | Likely-duplicate article pairs |
| `stubs` | Garbage/empty stub articles |
| `uncategorized` | Articles not in any taxonomy category |

## Alias System

Central to link resolution. `wiki/_meta/aliases.json` maps every known name to its canonical slug:

```json
{
  "参禅": "can-chan",
  "can-chan": "can-chan",
  "參禪": "can-chan",     // traditional Chinese (auto-generated)
  "Chan Practice": "can-chan"
}
```

Sources: article slug, title parts (split by `/`), simplified/traditional variants via OpenCC (if installed), merged_from history.

> **Note**: Simplified/traditional auto-conversion requires `opencc-python-reimplemented`. Install with `pip install opencc-python-reimplemented`. Without it, alias resolution still works but only matches exact text.
